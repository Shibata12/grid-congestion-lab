D-06: 最適化を「解く」──LP/MIPの直感とソルバー

D領域の本編は D-05: 総合演習 で一巡しました。ここからは発展編です。 D-03: SCUC/SCEDの概要 では、再給電を「目的関数＋制約条件」の最適化問題として 立てるところまでを学び、「アルゴリズムの詳細には踏み込まない」と預けていました。本ユニットがその回収です。 立てた問題を解くとはどういうことか——SCEDが速く解ける理由（LP）と、SCUCが重い理由（MIP）を、 2変数の例をグラフで解きながら、直感レベルで説明できるようになります。

このユニットで使うのは、次の3つの既習事項だけです：

  • D-03：SCUC/SCED ＝ 発電コスト最小化（目的関数）を、需給・潮流・出力上下限（制約条件）のもとで解く
  • D-03：SCUCは起動停止（0/1）、SCEDは出力配分（連続量）を決める
  • F-04: 中給システムの演算タイミングと時間断面：SCUCは早めに数回、SCEDは直前に多数回

範囲について：単体法・内点法・分枝限定法は「名前と発想」までで、アルゴリズムの中身や実装には踏み込みません。 市販・オープンソースのソルバーが何をしてくれるかの土地勘がつけば、本ユニットは合格です。

1. 「解く」とは何をすることか：LPとMIP

D-03で立てた問題を思い出します——「発電コスト min Σ ckPk を、 需給バランス・潮流制約・出力上下限のもとで最小にする」。 これを解くとは、制約をすべて満たし、かつコストが最小になる変数の値の組（P₁, P₂, …）を実際に求めることです。

このとき、変数の種類で問題の性格が大きく変わります。

  • SCED：変数は出力 Pk という連続量だけ。目的関数も制約もすべて1次式（線形）なので、 この形の問題を線形計画法Linear Programming（LP）。目的関数と制約条件がすべて1次式（線形）で、変数が連続量の最適化問題（およびその解法）。最適解は実行可能領域の頂点に現れ、単体法・内点法などで非常に大規模な問題も高速に解ける。SCEDはLPとして解ける。（LP）といいます。
  • SCUC：出力に加えて、各発電機の起動／停止 uk ∈ {0, 1} という 整数（0か1か）の変数が混ざります。連続量と整数が混在するこの形を 混合整数計画法Mixed Integer Programming（MIP）。連続量の変数と整数（0/1など）の変数が混在する最適化問題（およびその解法）。整数変数の組合せの数が爆発的に増えるため、LPよりはるかに計算が重い。SCUCは起動停止の0/1変数を含むMIP。（MIP）といいます。

結論を先に言うと——LPは「速い」、MIPは「重い」。 F-04で見た「SCEDは直前に多数回、SCUCは早めに数回」というタイミング設計は、まさにこの計算量の差の裏返しです。 なぜそうなるのかを、小さな例で体感しましょう。

2. 2変数のSCEDをグラフで解く

発電機が2台なら、紙の上で解けます。次の問題を考えます（数字はD-03の形式に合わせた例です）。

  • 発電機A：単価 8千円/MWh、出力 0〜150 MW ／ 発電機B：単価 12千円/MWh、出力 0〜150 MW
  • 需給バランス：PA + PB = 200（総需要 200 MW）
  • 潮流制約：線路Lに対して PTDFA = 0.6、PTDFB = 0.2、容量 88 MW

横軸に PA、縦軸に PB をとると、制約は1本ずつの直線になります。 すべての制約を同時に満たす点の集まりを実行可能領域最適化問題で、すべての制約条件を同時に満たす変数の値の組の集まり。LPでは直線（平面）で囲まれた多角形（多面体）になり、最適解はその頂点に現れる。といいます。 この例では、需給バランスが「線」なので、実行可能領域は需給の線の上の一部分（線分）です。

解き方はこうです。単価はAの方が安い（8 < 12）ので、Aをできるだけ増やしたい。 需給の線に沿ってAを増やしていくと、どこかで制約にぶつかります。Bの上限（PB ≤ 150、つまり PA ≥ 50）は 左端を決めるだけ。効いてくるのは潮流制約です—— 0.6PA + 0.2(200 − PA) ≤ 88 を整理すると PA ≤ 120。 よって最適解は PA = 120、PB = 80、 コストは 8×120 + 12×80 = 1,920 千円/時です。

ここで大事な観察が2つあります。第一に、最適解は実行可能領域（線分）の端＝角の点に来ました。 第二に、その角では潮流制約がぴったり等号で成立しています （0.6×120 + 0.2×80 = 88 ちょうど）。このように最適解で等号が成り立っている制約を 有効制約最適解において等号（ぴったり上限・下限）で成立している制約条件。バインディング制約ともいう。最適解の形とコストを実際に決めているのは有効制約であり、有効制約を少し緩めるとコストが下がる（→シャドープライス）。（バインディング制約）といいます。 答えを決めているのは有効制約です——もし線路Lの容量があと1MW広ければ、 安いAをもう少し使えてコストは下がったはず。この「制約1単位あたりの惜しさ」に値段をつけたものが、 次のユニットで学ぶシャドープライスです（予告）。

3. LPが速い理由：最適解は「角」にある

いまの観察は、2変数に限らないLPの一般的な性質です。 変数が増えると、実行可能領域は直線・平面で囲まれた多角形（多面体）になります。 目的関数は1次式＝「平らな坂」なので、坂を下りきった先は必ず多面体の頂点（角）です （同率で最適な辺や面が現れることはあっても、頂点より内側だけが最適になることはありません）。

つまりLPでは、無限にある実行可能な点のうち、頂点だけを調べれば良い。 この性質を利用して、頂点から隣の「より良い頂点」へ賢く渡り歩くのが単体法（シンプレックス法）、 領域の内側を最短距離で突っ切るように近づくのが内点法です（本サイトでは名前まで）。 どちらも成熟した技術で、変数が数十万〜数百万あるLPでも実用時間で解けます。 SCEDが「直前に・多数回」実行できるのは、LPのこの速さがあるからです。

4. 0/1が入ると難しい：組合せ爆発と分枝限定法

ではSCUCはなぜ重いのか。起動停止 uk は 0 か 1 のどちらかで、中間がありません。 「0.4だけ起動する」は許されないのです。すると原理的には、全発電機のオン・オフの組み合わせを考えることになります。 発電機が30台なら 230 ≒ 約10億通り。1コマではなく48コマ分をまとめて決めるので、 組み合わせはさらに膨れ上がります。これが組合せ爆発で、しらみつぶしは不可能です。

そこで使われる代表的な発想が分枝限定法です。

  • 緩和：まず 0/1 の条件を「0〜1の小数でよい」ことにして、LPとして解いてみる（速い）。
  • 分枝：小数になった変数（例：u₃ = 0.4）について、「u₃ = 0 とした問題」と「u₃ = 1 とした問題」に枝分かれさせて、それぞれまた解く。
  • 限定：枝の途中で「この先をどう決めても、今見つかっている答えより良くならない」と分かった枝は、丸ごと切り捨てる。

切り捨てがうまく効けば現実的な時間で解けますが、それでもLPを何百回・何千回も解くことに変わりはなく、 MIPはLPよりはるかに重い。しかも実務では「厳密な最適でなくても、最適との差が0.1%以内なら良い」といった 妥協点（ギャップ）を設けて計算を打ち切ることもあります。 SCUCが前日など早めのタイミングで、回数を絞って実行されるのは、この重さゆえです。

5. ソルバーと実務：SEは何をするのか

ここまでの解法——単体法・内点法・分枝限定法——を、システム開発のたびに自作するわけではありません。 LP/MIPを解く技術はソルバーLP・MIPなどの最適化問題を解く既製のソフトウェアエンジン。モデル（目的関数・制約条件）とデータを渡すと最適解を返す。商用・オープンソースの汎用製品が存在し、SCUC/SCEDのようなシステムはソルバーを部品として組み込んで作られる。と呼ばれる 既製のエンジン（商用・オープンソースの汎用製品）に集約されており、 SCUC/SCEDのようなシステムは、ソルバーを部品として組み込んで作られます。

だからシステムに関わるエンジニアの仕事の中心は、アルゴリズムの発明ではなく——

  • モデルを正しく立てる：D-03の目的関数・制約条件を、対象の系統・制度に合わせて過不足なく数式化する。
  • 入力データを整える：系統データ（C-02）から作るPTDF、発電機の単価・上下限、需要と再エネの予測（F-07）——入力が違えば最適解も違う。
  • 計算時間と品質を管理する：ゲートクローズまでに解が出るか（F-04）、ギャップはどこまで許すか、解けなかったときのフォールバックはあるか。

「最適化は魔法の箱ではなく、モデルとデータの正しさがすべて」——これが本ユニットの締めくくりであり、 D領域の計算たち（PTDF・再給電・SCUC/SCED）が実システムで動くときの現場感覚です。 有効制約に「値段」をつける続きの話は、D-07: 混雑の値段──シャドープライスとLMP で扱います。
