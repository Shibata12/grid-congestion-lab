C-02: 系統データと混雑計算のインプット

C-01: 母線・送電線・系統モデルの構造 では、系統を 母線（節点）・送電線（枝）・アドミタンス行列（Yバス）で表す「器」を作りました。 本ユニットでは、その器に実際に入る「データ」を扱います。 F-02: 次期中給システムの機能構成 で出てきた 3つの計画機能（系統構成計画・作業停止計画・需給計画）が、系統モデルのどこに反映されるのかを見ていきます。

このユニットの範囲について： 本ユニットは、公開情報で裏付けられる範囲で「計画データが系統モデルにどう対応するか」を扱います。 各計画の提出手続きや設備停止の概念はOCCTO等の公開資料に基づき、開閉器の状態管理や需給計画機能の 内部ロジックには踏み込みません。参考にした公開資料は末尾の参考資料を参照してください。

1. 系統モデルに入力するデータ

C-01で作ったのは、系統の「かたち」を表す器でした。しかし器だけでは計算できません。 実際の運用では、いつ・どんな構成で・どれだけ発電し消費するかというデータが入って、 はじめて潮流計算（A-02）や混雑計算（領域B・D）が動きます。

F-02で見た3つの計画機能が、まさにこの入力データにあたります。 送配電網協議会が次世代電力系統ワーキンググループで公開した機能構成でも、需給計画・系統構成計画・作業停止計画が 最適化（SCUC／SCED）の上流に位置づけられていました（末尾の参考資料）。 本ユニットでは、それぞれが系統モデルのどこに反映されるかを整理します。

2. 系統構成計画とトポロジー

系統構成計画系統をどう構成するか（どの母線とどの送電線がつながっているか＝トポロジー）を定める計画。系統モデルの構造を決める。は、系統を どう構成するかを定めます。本ユニットで注目するのは、その結果として決まる トポロジー、すなわちどの母線とどの送電線がつながっているかです。

トポロジーが決まると、系統モデルの構造が決まります。C-01で見たように、アドミタンス行列（Yバス）の 非対角要素は「2つの母線を結ぶ線路があれば −y、なければ 0」でした。つまり トポロジー（線路のつながり方）が、Yバスのどこに値が入るか（0 か非 0 か）を決めるのです。

送電線が系統に入る・出るの切り替えは、変電所の 開閉器系統の接続構成を変更するための装置。遮断器・断路器など。（遮断器・断路器）の入／切で行われます。 本ユニットでは、開閉器の具体的な運用手順や状態管理の詳細には立ち入らず、 「トポロジーが系統モデルの構造（Yバスの形）を決める」という対応関係に焦点を当てます。

3. 作業停止計画

送電線や変圧器などの設備は、保守・点検のために計画的に停止させる必要があります。これを定めるのが 作業停止計画保守・点検などで設備を停止する計画。停止中の設備は系統モデルから除いて扱う。OCCTOが調整・記載要領を公開している。です。 OCCTOは、事業者間および一般送配電事業者との調整を経た作業停止計画を広域機関システムで扱い、 作業停止計画の調整マニュアルや記載要領を公開しています（末尾の参考資料）。

停止した送電線は、その期間は「ない」ものとして系統モデルに反映します。 やることはC-01の組み立てルールの逆操作です。停止する線路について、

  • その線路がつながっていた2つの母線の対角から、その線路のアドミタンスを引く
  • その2母線間の非対角を 0 にする

こうしてYバスからその線路を除きます。設備が止まると系統の流れ方（潮流分布）が変わり、 別の送電線に潮流が集中して混雑が起きやすくなることもあります。 だから作業停止計画は、混雑管理の重要な前提条件になります。 （具体的な操作は §5 の例で確認します。）

4. 需給計画と各母線の注入

需給計画各母線（エリア）の発電量・負荷量の計画。発電計画値・需要計画値からなり、系統モデルでは各母線の注入として入力される。は、 各母線（エリア）でどれだけ発電し、どれだけ消費するかの計画です。 これは新しい数字ではなく、E-01: BGと発電計画値 で学んだ 発電計画値（発電側）と需要計画値（負荷側）からなります。 各事業者は電気事業法等に基づき、これらの計画を広域機関（OCCTO）のシステムを通じて提出します（末尾の参考資料）。

系統モデルでは、各母線の発電量・負荷量は、A-02で学んだ注入として入ります。

発電が多い母線は ＋（系統へ送り出す）、負荷が多い母線は −（系統から引き込む）になります。 こうして「Yバス（構成）」と「各母線の注入（需給）」がそろって、潮流計算の入力が完成します。 C-01の器に、系統構成・作業停止でかたちを整え、需給計画で中身（注入）を入れる、というイメージです。

5. N-1解析時のモデル

B-01: 混雑の判定とN-1基準 では、 設備が1つ止まっても過負荷にならないかを評価するN-1基準を学びました。 このN-1解析では、系統モデルを想定故障ごとに「その設備（たとえば1回線）を落とした構成」に作り替えます。 やること自体は作業停止（§3）と同じで、その線路のアドミタンスをYバスから除く操作です。 違いは目的で、作業停止は「実際に止める計画」、N-1は「仮に止まったらどうなるかを評価する想定」です。

例で見ましょう。母線 1・2・3 が三角形（ループ）につながり、線路1-2のアドミタンスが y₁₂ = 5、 線路2-3が y₂₃ = 3、線路1-3が y₁₃ = 2 だとします。C-01のルールで組み立てた基準構成のYバスは次のとおりです。

ここで線路1-3（y₁₃ = 2）が1回線開放（N-1）されたとします。 線路1-3を除くので、母線1と母線3の対角からそれぞれ 2 を引き（7→5、5→3）、母線1-3間の非対角を 0 にします。 N-1後のYバスはこうなります。

ループが1本切れてパスになりましたが、母線1と母線3は母線2を経由してまだつながっています。 このように1回線が止まっても系統が維持される（過負荷にならない）かを確認するのがN-1基準でした。 各想定故障についてこのようにYバスを作り直し、潮流を再計算して過負荷の有無を調べるのが N-1解析の中身であり、領域Dの混雑計算 （このチェックを高速に回す方法は D-04: 想定故障計算とLODF）へとつながっていきます。
