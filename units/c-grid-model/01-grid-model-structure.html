C-01: 母線・送電線・系統モデルの構造

A領域では、系統を物理法則（キルヒホッフ・潮流・インピーダンス）として学びました。 実際の系統を計算機で扱うには、現実の設備を計算できる「モデル」に置き換える必要があります。 本ユニットでは、母線・送電線をどうデータ化し、系統全体を アドミタンス行列としてどう組み立てるかを、構造と意味の理解に重点を置いて学びます。 ここで作るモデルが、領域Dの混雑計算の入力になります。

このユニットの範囲について： 本ユニットはモデルの構造と意味の理解を重視し、複素数や大規模な行列計算には踏み込みません。 アドミタンスは“流れやすさ”の実数の大きさとして扱います。 §3の基幹系統・上位2電圧は公開資料（OCCTO）に基づきます。参考にした公開資料は末尾の参考資料を参照してください。

1. 系統をモデル化するとは

現実の電力系統は、発電所・変電所・送電線・鉄塔など、おびただしい数の設備でできています。 これをそのまま計算することはできません。そこで、計算できる形に抽象化します。 A-01: オームの法則とキルヒホッフの法則 で扱った回路の考え方を思い出してください。 回路は節点（ノード）と、それをつなぐ枝（ブランチ）でできていました。系統も同じように捉えます。

  • 母線（バス）＝節点：設備が集まって接続する点
  • 送電線・変圧器＝枝：母線と母線をつなぐもの

こうして現実の系統を「母線（節点）と枝のつながり」として表したものが系統モデルです。 なぜモデル化するかというと、A-02: 潮流計算の基礎 の潮流計算や、 領域B・Dの混雑計算の入力にするためです。モデルがなければ、計算機は系統を扱えません。

2. 母線とは何か

母線複数の設備が接続する接続点。回路でいう節点（ノード）に相当する。物理的には変電所内の導体（ブスバー）。（バス）とは、 発電機・送電線・変圧器などが集まって接続する接続点のことです。回路でいう節点（ノード）に相当し、 A-02: 潮流計算の基礎 でも「母線」として登場していました。

物理的な実体は、変電所の中にある「ブスバー（母線）」と呼ばれる金属の導体です。 ここに送電線・変圧器・発電機などがつながります。1つの変電所には、 電圧階級ごとに複数の母線があるのがふつうです（たとえば 500kV の母線と 275kV の母線が、変圧器でつながっています）。

系統モデルでは、この母線1つを1つの節点として扱い、母線1・母線2…と番号をつけて区別します。 あとで見るアドミタンス行列は、この母線（節点）を行と列に並べたものになります。

3. 基幹系統の規模感と上位2電圧

電力系統は電圧の階層構造になっています。発電所から超高圧で送り出し、変電所で段階的に電圧を下げ、 最後は配電線で家庭や工場へ届けます。なぜ高い電圧で送るのかというと、 同じ電力 P = V × I を送るとき、電圧 V を高くするほど電流 I が小さくなり、 送電損失（電流の2乗に比例）を小さくできるからです。だから大容量・長距離の送電ほど高い電圧を使います。

このうち、系統の骨格を成す最上位の送電網を基幹系統各エリアの上位2電圧（最上位電圧とその1階級下）の送電線・母線。系統の骨格をなし、混雑管理の主舞台になる。といいます。 OCCTO（電力広域的運営推進機関）は基幹系統を、 各エリアの上位2電圧（最上位電圧と、その1階級下の電圧。最上位が250kV未満の区域ではその最上位電圧）の 送電線・母線として定義しています。具体的な電圧はエリアで異なり、たとえば 500kV と 275kV、地域によっては 275kV と 154kV（または187kV）などです。

基幹系統だけでも母線や送電線の数は多く、想定故障（N-1）の組み合わせも膨大です。 だからこそ系統をモデル化して計算機で扱う必要があり、系統混雑管理（領域B・D・F）の主な舞台も、この上位2電圧の基幹系統になります。

4. 送電線データとアドミタンス行列

いよいよモデルの中身です。まず、各送電線を A-03: インピーダンスとリアクタンス で学んだ インピーダンス Z = R + jX（線路ごとの R・X の値）でデータ化します。 そして、その逆数を考えます。インピーダンスが“流れにくさ”なら、その逆数は“流れやすさ”で、これを アドミタンスインピーダンスの逆数。電流の“流れやすさ”を表す。記号 Y、Y=1/Z。 Y といいます。

送電線では抵抗よりリアクタンスがずっと大きい（X ≫ R、A-03）ので、本ユニットではアドミタンスを y = 1/X の実数の“流れやすさ”として扱い、複素数には立ち入りません。 たとえばリアクタンスが小さい（流れやすい）線路ほど、アドミタンス Y は大きくなります。

この線路ごとのアドミタンスを使って、系統全体を1つの正方行列にまとめたものが アドミタンス行列系統全体のアドミタンスを母線×母線の正方行列にまとめたもの。Yバス（Y-bus）ともいう。潮流計算やPTDFの計算に使う。（Yバス）です。 母線を行と列に並べ、次のルールで組み立てます。

  • 対角 Yii：母線 i につながるすべての線路のアドミタンスの合計
  • 非対角 Yij：母線 i と母線 j を結ぶ線路のアドミタンス ×（−1）。線路がなければ 0

小さな例で組み立ててみましょう。母線 1 ─ 2 ─ 3 が一直線につながり（母線1と母線3の間に直接の線路はない）、 線路1-2のアドミタンスが y₁₂ = 5、線路2-3が y₂₃ = 4 だとします。

ルールに従って組み立てると、Yバスは次のようになります。

中身を確かめましょう。対角は合計です。母線1は線路1-2だけなので Y₁₁ = 5、 母線2は線路1-2と2-3の2本なので Y₂₂ = 5 + 4 = 9、母線3は線路2-3だけなので Y₃₃ = 4。 非対角は線路アドミタンスに −1 を掛けたもので、Y₁₂ = Y₂₁ = −5、Y₂₃ = Y₃₂ = −4。 そして母線1と母線3は直接つながっていないので Y₁₃ = Y₃₁ = 0 です。 できあがった行列が対称になること、0 の位置が「直接の線路がない母線ペア」を表すことに注目してください。

このアドミタンス行列こそ、A-02 の潮流計算が内部で使っているものです。 各母線の注入が与えられると、このYバスを通じて各母線の電圧と各送電線の潮流が定まります。 そして D-01: PTDFと感度分析 で「PTDFは系統全体のリアクタンス（アドミタンス行列）から計算される」と述べていたのは、 まさにこのYバスのことでした。C領域で作るモデルが、D領域の混雑計算の入力になるのです。
